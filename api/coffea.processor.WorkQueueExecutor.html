Parameters:

  • items (sequence or generator) ‚Äì Sequence of input arguments

  • function (callable) ‚Äì A function to be called on each input, which returns an accumulator instance

  • accumulator (Accumulatable) ‚Äì An accumulator to collect the output of the function

  • status (bool) ‚Äì If true (default), enable progress bar

  • unit (str) ‚Äì Label of progress bar unit

  • desc (str) ‚Äì Label of progress bar description

  • compression (int, optional) ‚Äì Compress accumulator outputs in flight with LZ4, at level specified (default 9) None` sets level to 1 (minimal compression)

  • options (# work queue specific)

  • cores (int) ‚Äì Maximum number of cores for work queue task. If unset, use a whole worker.

  • memory (int) ‚Äì Maximum amount of memory (in MB) for work queue task. If unset, use a whole worker.

  • disk (int) ‚Äì Maximum amount of disk space (in MB) for work queue task. If unset, use a whole worker.

  • gpus (int) ‚Äì Number of GPUs to allocate to each task. If unset, use zero.

  • resource_monitor (str) ‚Äì

    If given, one of ‚Äòoff‚Äô, ‚Äòmeasure‚Äô, or ‚Äòwatchdog‚Äô. Default is ‚Äòoff‚Äô. - ‚Äòoff‚Äô: turns off resource monitoring. Overriden to ‚Äòwatchdog‚Äô if resources_mode

    is not set to ‚Äòfixed‚Äô.

    • ‚Äômeasure‚Äô: turns on resource monitoring for Work Queue. The

      resources used per task are measured.

    • ‚Äôwatchdog‚Äô: in addition to measuring resources, tasks are terminated if they

      go above the cores, memory, or disk specified.

  • resources_mode (str) ‚Äì

    one of ‚Äòfixed‚Äô, ‚Äòmax-seen‚Äô, or ‚Äòmax-throughput‚Äô. Default is ‚Äòmax-seen‚Äô. Sets the strategy to automatically allocate resources to tasks. - ‚Äòfixed‚Äô: allocate cores, memory, and disk specified for each task. - ‚Äòmax-seen‚Äô or ‚Äòauto‚Äô: use the cores, memory, and disk given as maximum values to allocate,

    but first try each task by allocating the maximum values seen. Leads to a good compromise between parallelism and number of retries.

    • ‚Äômax-throughput‚Äô: Like max-seen, but first tries the task with an

      allocation that maximizes overall throughput.

    If resources_mode is other than ‚Äòfixed‚Äô, preprocessing and accumulation tasks always use the ‚Äòmax-seen‚Äô strategy, as the former tasks always use the same resources, the latter has a distribution of resources that increases over time.

  • split_on_exhaustion (bool) ‚Äì Whether to split a processing task in half according to its chunksize when it exhausts its the cores, memory, or disk allocated to it. If False, a task that exhausts resources permanently fails. Default is True.

  • fast_terminate_workers (int) ‚Äì Terminate workers on which tasks have been running longer than average. The time limit is computed by multiplying the average runtime of tasks by the value of ‚Äòfast_terminate_workers‚Äô. Since there are legitimately slow tasks, no task may trigger fast termination in two distinct workers. Less than 1 disables it.

  • manager_name (str) ‚Äì Name to refer to this work queue manager. Sets port to 0 (any available port) if port not given.

  • port (int or tuple(int, int)) ‚Äì Port number or range (inclusive of ports )for work queue manager program. Defaults to 9123 if manager_name not given.

  • password_file (str) ‚Äì Location of a file containing a password used to authenticate workers.

  • ssl (bool or tuple(str, str)) ‚Äì Enable ssl encryption between manager and workers. If a tuple, then it should be of the form (key, cert), where key and cert are paths to the files containing the key and certificate in pem format. If True, auto-signed temporary key and cert are generated for the session.

  • extra_input_files (list) ‚Äì A list of files in the current working directory to send along with each task. Useful for small custom libraries and configuration files needed by the processor.

  • x509_proxy (str) ‚Äì Path to the X509 user proxy. If None (the default), use the value of the environment variable X509_USER_PROXY, or fallback to the file /tmp/x509up_u${UID} if exists. If False, disables the default behavior and no proxy is sent.

  • environment_file (optional, str) ‚Äì Conda python environment tarball to use. If not given, assume that the python environment is already setup at the execution site.

  • wrapper (str) ‚Äì Wrapper script to run/open python environment tarball. Defaults to python_package_run found in PATH.

  • treereduction (int) ‚Äì Number of processed chunks per accumulation task. Defaults is 20.

  • verbose (bool) ‚Äì If true, emit a message on each task submission and completion. Default is false.

  • print_stdout (bool) ‚Äì If true (default), print the standard output of work queue task on completion.

  • debug_log (str) ‚Äì Filename for debug output

  • stats_log (str) ‚Äì Filename for tasks statistics output

  • transactions_log (str) ‚Äì Filename for tasks lifetime reports output

  • tasks_accum_log (str) ‚Äì Filename for the log of tasks that have been processed and accumulated.

  • filepath (str) ‚Äì Path to the parent directory where to create the staging directory. Default is ‚Äú.‚Äù (current working directory).

  • custom_init (function, optional) ‚Äì A function that takes as an argument the queue‚Äôs WorkQueue object. The function is called just before the first work unit is submitted to the queue.
